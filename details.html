How I built it

This started as a simple question: can I use machine learning to predict tomorrow's weather at SFO?

Question

I wanted to put the ML concepts I learned at Berkeley into a real project, so I chose a target that was concrete, measurable, and easy to explain: tomorrow's highest temperature.

Experiment

I cleaned NOAA station history, built features, trained XGBoost and LSTM models, tuned them, and compared them on held-out dates instead of relying on one good-looking chart.

Result

The LSTM performed better for daily highs because it can learn the recent weather sequence, but the final predictor still respects live forecasts from professional weather stations.

Automation

The project now runs on AWS Lightsail. A small Ubuntu server refreshes Google Weather, NWS, Open-Meteo, and KSFO observations on a schedule, rebuilds this page, and pushes the static site to GitHub Pages.

Hardest part: defining the target correctly. Predicting a random timestamp is different from predicting the next local calendar-day high, and getting that alignment right made the evaluation honest.

Automation stack: Lightsail systemd timers fetch the paid Google Weather API within a strict event budget, update the local SQLite archive, run the blend/scoring scripts, and publish the latest dashboard through a GitHub deploy key. The Mac no longer has to stay awake for the forecast to update.

Future work: collect more scored refreshes, add more Bay Area station history, and tune the blend weights only after the archive shows which sources are consistently closest to the NWS high.

LSTM vs XGBoost

The comparison matters because XGBoost is a strong tabular baseline. The LSTM only earns the headline because it improves tomorrow's daily-high error on held-out days.

Why the LSTM wins hereThe target is a daily high, so the recent shape of the temperature curve matters. The LSTM reads the 48-hour sequence directly instead of depending only on engineered lag columns.
Why XGBoost still mattersXGBoost is fast, interpretable, and competitive. Its feature-importance plot shows recent temperature history is the strongest tabular signal.
What the test provesOn 475 held-out forecast days, LSTM lowers MAE from 3.87°F to 3.19°F, a 18% error reduction.

Held-Out Forecasts

Daily-high predictions on dates the model did not see during training.

The model is strongest on ordinary SFO weather and weakest on rare hot days. That failure mode is useful because it points directly to the next improvement: more hot-day examples or stronger extreme-day weighting.
